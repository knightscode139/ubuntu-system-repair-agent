# Ubuntu Solutions Synthetic Dataset

This directory contains a synthetic dataset of Ubuntu system repair solutions and a ChromaDB vector database for semantic search.

## Dataset Overview

- **File**: `solutions.json`
- **Total Solutions**: 150
- **Generated by**: Claude Sonnet
- **Purpose**: Provide repair solutions for common Ubuntu system errors

## Categories

- **Package Management** (38 solutions)
- **Disk/Filesystem** (32 solutions)
- **Systemd Services** (33 solutions)
- **Network** (31 solutions)
- **Boot/GRUB** (3 solutions)
- **GPU/Drivers** (4 solutions)
- **Performance** (5 solutions)
- **User/Permissions** (4 solutions)

## Solution Schema

Each solution contains:
- `category`: Error category
- `error_trigger`: Error message or symptom
- `context`: When/why the error occurs
- `solution_explanation`: Brief explanation of the fix
- `steps`: Array of repair steps with commands
- `risk_level`: Critical, High, Medium, or Low
- `backup_required`: Boolean flag
- `requires_reboot`: Boolean flag
- `validation_command`: Command to verify the fix
- `alternative_solution`: Alternative approach if available

## ChromaDB Vector Database

The `chroma_db/` directory contains a pre-built vector database for semantic similarity search.

### Rebuilding the Database

To rebuild the ChromaDB from the JSON dataset:

```bash
uv run python scripts/build-chroma-db.py
```

This creates embeddings from:
- Error trigger message
- Context information
- Solution explanation

## Usage

The vector database enables finding relevant solutions by querying with error messages or symptoms, even if they don't match exactly.
